시작하기

PhotoDump는 무엇을 하나요?

PhotoDump는 라이브 포토를 포함한 사진과 동영상을 원본 화질 그대로 iPhone에서 사용자가 직접 관리하는 대상(USB 또는 SSD 드라이브, 직접 운영하는 S3 호환 스토리지, 또는 사용자가 운영하는 REST 서버)으로 복사합니다.

어떤 저장 대상이 지원되나요?

USB/SSD 드라이브, S3 호환 오브젝트 스토리지(AWS S3, MinIO 및 호환 서비스), 그리고 사용자가 직접 관리하는 REST 서버를 지원합니다. 현재 가장 안정적인 옵션은 USB/SSD이며, S3와 REST는 베타로 표시되어 있습니다.

계정을 만들어야 하나요?

아니요. 가입, 로그인, PhotoDump 계정 같은 것은 전혀 없습니다. 사용하려는 저장 대상만 설정하면 됩니다.

구독이나 인앱 구매가 있나요?

정기 구독은 없습니다. PhotoDump는 한 번만 구매하면 베타 기능을 포함한 모든 저장 대상을 사용할 수 있는 단발성 구매 앱입니다.

어떤 iOS 버전이 필요한가요?

iOS 16 이상이 필요합니다. 잠금 화면의 라이브 액티비티와 같은 일부 기능은 더 최신 iOS 버전이 설치되어 있을 때 추가로 활용됩니다.

PhotoDump는 어떤 언어를 지원하나요?

영어, 러시아어, 독일어, 프랑스어, 스페인어, 포르투갈어(브라질), 중국어(간체), 일본어, 한국어, 아랍어, 카자흐어를 지원합니다. 앱은 기기 언어를 자동으로 따르며, 설정에서 직접 언어를 선택할 수도 있습니다.

USB 및 SSD

USB나 SSD 드라이브는 어떻게 연결하나요?

드라이브를 iPhone에 (직접 또는 어댑터를 통해) 연결한 다음 PhotoDump를 열고 저장 대상으로 "USB 드라이브"를 선택하세요. 메시지가 표시되면 폴더 접근 권한을 허용하세요.

USB/SSD는 정말로 100% 로컬로 동작하고 인터넷이 필요 없나요?

네. 파일은 사진 보관함에서 드라이브로 직접 전송됩니다. 어디에도 업로드되지 않으며, 전송 자체에는 인터넷 연결이 필요하지 않습니다(단, 기기가 아니라 iCloud에만 저장되어 있는 원본을 다운로드할 때는 예외입니다).

하나의 드라이브를 여러 대의 iPhone에서 사용할 수 있나요?

네. 설정할 때 기기마다 서로 다른 대상 폴더 이름을 지정하면 각 기기의 기록이 분리되어 서로 다른 휴대폰의 파일이 섞이지 않습니다.

마이그레이션 도중 드라이브 연결이 끊기면 어떻게 되나요?

마이그레이션이 자동으로 일시 중지됩니다. 드라이브를 다시 연결하면 중단된 지점부터 정확히 이어서 진행되며, 다시 복사되거나 손실되는 파일은 없습니다.

앱이 왜 폴더 이름을 물어보나요?

폴더 이름은 드라이브에서 파일(및 PhotoDump의 내부 기록 데이터베이스)이 저장되는 위치를 결정합니다. 이를 통해 하나의 드라이브를 공유하는 여러 기기의 파일이 서로 섞이지 않도록 합니다.

PhotoDump가 드라이브를 포맷하거나 지우나요?

아니요. PhotoDump는 사용자가 선택한 대상 폴더에만 파일을 씁니다. 드라이브를 포맷하거나 지우거나, 그 외의 다른 부분을 건드리는 일은 절대 없습니다.

S3 및 REST (베타)

S3 호환 저장 대상은 어떻게 설정하나요?

저장 대상으로 "S3 스토리지"를 선택하고 버킷 이름, 리전, 액세스 키, 시크릿 키를 입력하세요. AWS뿐만 아니라 S3 호환 제공업체라면 어디든 사용할 수 있습니다.

Amazon S3만 지원되나요?

아니요. S3 API를 지원하는 스토리지 서비스라면 MinIO 같은 셀프 호스팅 옵션을 포함해 모두 동일하게 사용할 수 있습니다.

REST 서버 대상은 어떻게 설정하나요?

저장 대상으로 "REST API"를 선택하고 서버의 업로드 URL과 필요한 인증 헤더를 입력하세요. PhotoDump가 전송하는 정확한 요청 형식은 앱 내 설정 가이드를 참고하세요.

S3와 REST는 왜 베타로 표시되어 있나요?

USB/SSD 지원보다 나중에 추가된 기능으로, 네트워크 중단이나 제공업체별 특이 사항 같은 예외 상황에 대한 안정화 작업이 아직 진행 중입니다. 사용하기에 안전하지만, 현재 가장 검증이 많이 된 방식은 USB/SSD입니다.

S3 액세스 키는 안전하게 저장되나요?

네. 자격 증명은 iOS가 비밀번호 저장에 사용하는 것과 동일한 보안 저장소인 기기의 키체인에 저장되며, 사용자가 설정한 대상 이외의 어느 곳으로도 전송되지 않습니다.

마이그레이션

"삭제해도 안전함" 기능은 어떻게 동작하나요?

파일은 PhotoDump가 대상 위치에 해당 파일이 존재함을 확인한 후에만 휴대폰에서 삭제해도 안전한 것으로 표시됩니다. 이 확인이 통과되기 전에는 어떤 파일도 삭제 대상으로 제안되지 않습니다.

마이그레이션을 일시 중지했다가 다시 시작할 수 있나요?

네, 언제든지 가능합니다. 앱을 닫았다가 나중에 다시 열어도 처음부터 다시 시작하지 않고 중단된 지점부터 정확히 이어서 진행됩니다.

마이그레이션이 느리거나 iCloud에서 다운로드하는 중에 멈추는 이유는 무엇인가요?

사진이나 동영상이 기기가 아니라 iCloud에만 저장되어 있는 경우, 다운로드 속도는 Apple의 iCloud 서비스가 해당 계정에 허용하는 속도에 따라 달라집니다. 이는 PhotoDump가 아니라 Apple이 제어하는 부분입니다. 파일이 한 번 다운로드되면 더 이상 iCloud에 의존하지 않습니다.

마이그레이션 중에 휴대폰이 잠기면 어떻게 되나요?

마이그레이션 화면이 열려 있는 동안에는 PhotoDump가 화면이 자동으로 잠기지 않도록 방지하므로, 밝기를 낮춰도 안전합니다. 수동으로 휴대폰을 잠그면 마이그레이션이 일시 중지되며, 잠금을 해제하면 다시 시작됩니다.

마이그레이션이 백그라운드에서 실행되나요?

iOS는 모든 앱의 백그라운드 사진 업로드를 제한하므로, PhotoDump는 앱을 열어둔 상태에서 가장 잘 동작합니다. 잠금 화면에서는 라이브 액티비티로 진행 상황을 표시하며, 백그라운드 실행 시간이 끝나면 앱을 다시 열도록 안내합니다.

파일 마이그레이션이 실패하면 어떻게 되나요?

PhotoDump는 실패로 표시하기 전에 몇 차례 자동으로 재시도합니다. 실패한 항목은 마이그레이션 로그에 표시되며, 언제든지 수동으로 다시 시도할 수 있습니다.

PhotoDump가 사진을 자동으로 삭제하는 경우가 있나요?

아니요. PhotoDump는 파일을 삭제해도 안전한 것으로 표시할 뿐, 사진 보관함에서 스스로 무언가를 삭제하는 일은 절대 없습니다. 삭제는 항상 사용자가 직접 수행하는 수동 작업입니다.

마이그레이션할 앨범이나 사진을 선택할 수 있나요?

네. 전체 보관함을 마이그레이션하거나, 마이그레이션 설정 화면에서 특정 프리셋과 앨범을 선택할 수 있습니다.

탐색 및 검색

사진 앱처럼 저장 대상을 탐색할 수 있나요?

네. 앨범, 전체 화면 보기, 검색 모두 파일이 아직 휴대폰에 있든 이미 대상으로 마이그레이션되었든 동일하게 동작합니다.

검색은 어떻게 동작하나요?

PhotoDump는 각 사진에 담긴 내용(인물, 장소, 사물)과 촬영 위치를 기기에서 전적으로 인덱싱하므로, 정확한 파일 이름이나 날짜를 입력하지 않고도 내용이나 위치로 검색할 수 있습니다.

드라이브가 연결되어 있지 않을 때도 검색이 되나요?

원본 드라이브에 현재 접근할 수 없는 경우 검색 태그는 숨겨지므로, 지금 실제로 열 수 없는 콘텐츠를 가리키는 검색 결과가 표시되는 일은 없습니다.

저장공간 및 캐시

PhotoDump가 휴대폰의 저장공간을 추가로 사용하나요?

의미 있는 수준으로는 사용하지 않습니다. PhotoDump는 마이그레이션된 파일의 사본을 휴대폰 저장공간에 별도로 보관하지 않습니다. 전송 중 사용되는 작은 임시 파일은 자동으로 정리됩니다.

설정의 "캐시 지우기"는 무슨 기능인가요?

중단된 마이그레이션이나 동영상 미리보기로 남은 썸네일과 임시 파일을 지워, 자동 정리를 기다리지 않고 즉시 공간을 회수합니다.

USB/SSD와 S3/REST의 저장공간 사용 방식이 다른가요?

USB/SSD는 파일을 휴대폰의 로컬 저장공간에 전혀 복사하지 않고 드라이브로 직접 스트리밍합니다. S3/REST는 업로드하는 동안 파일을 잠깐 로컬에 보관했다가 자동으로 제거합니다.

개인정보

PhotoDump가 분석 데이터나 추적 데이터를 수집하나요?

아니요. 앱에는 분석, 충돌 보고, 광고 SDK가 전혀 포함되어 있지 않습니다. 자세한 내용은 개인정보 처리방침을 참고하세요.

누가 내 사진을 볼 수 있나요?

사용자 본인뿐입니다. 파일은 사용자의 기기에서 설정한 대상으로 직접 전송됩니다. PhotoDump 개발자는 마이그레이션한 어떤 것의 사본도 절대 받지 않습니다.
